# Contributing to the Voltex Network

Thank you for your interest in contributing. This document covers the profile repository; for contributions to specific projects (AZRAEL, FCRI, VoltexBazar, etc.) refer to the CONTRIBUTING.md in those respective repositories.

## How to Contribute to This Repository

This is a GitHub profile repository (`Davidcarmelalex/Davidcarmelalex`). Contributions may include:

- Fixing broken links or outdated project descriptions
- Improving documentation in `docs/`
- Suggesting new ecosystem additions via issues
- Improving architecture diagrams or the ASGARDIA master architecture

## Development Workflow

1. **Fork** the repository
2. **Create a branch** from `main`:
   ```
   git checkout -b fix/your-description
   ```
3. **Make your changes** — keep them focused and minimal
4. **Commit** with a clear message following [Conventional Commits](https://www.conventionalcommits.org/):
   ```
   docs: fix broken link to VoltexBazar
   chore: update .gitignore patterns
   feat: add HermesLock to ecosystem table
   ```
5. **Push** and open a Pull Request against `main`

## Commit Message Format

```
<type>: <short description>

[optional body]
```

Types: `feat`, `fix`, `docs`, `chore`, `refactor`, `style`, `test`

## Standards

- Keep the README visually clean — no unnecessary badges or noise
- All links must be live and verified before submitting
- Architecture documents must remain consistent with the README system diagram
- No promotional or commercial content in `docs/`

## Code of Conduct

All contributors are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md).

## Questions

Open a GitHub Discussion or reach out via the contact information in the README.
